Политика конфиденциальности

ВАЖНО: Система семейной безопасности ALADDIN спроектирована для работы БЕЗ сбора персональных данных. Мы обеспечиваем полную функциональность защиты семьи, полностью исключая сбор, обработку и хранение ваших персональных данных.

1. Общие положения

Настоящая политика обработки персональных данных составлена в соответствии с требованиями Федерального закона от 27.07.2006 № 152-ФЗ «О персональных данных» и определяет порядок обработки персональных данных и меры по обеспечению безопасности персональных данных, предпринимаемые системой семейной безопасности ALADDIN.

Оператор ставит своей важнейшей целью и условием осуществления своей деятельности полное исключение сбора, обработки и хранения персональных данных пользователей, обеспечивая при этом полную функциональность системы защиты семьи от киберугроз.

2. Основные понятия

Анонимная сессия безопасности

Уникальный идентификатор сессии защиты, генерируемый системой без привязки к личности пользователя.

Анонимный профиль безопасности

Профиль пользователя системы безопасности, содержащий только роли (родитель/ребенок/пожилой), возрастные группы и типы устройств без персональной информации.

Обезличенные данные безопасности

Данные о сессиях защиты, которые не позволяют определить конкретного пользователя без использования дополнительной информации, недоступной Оператору.

Локальная аутентификация

Процесс подтверждения личности пользователя, осуществляемый исключительно на устройстве пользователя без передачи данных на серверы Оператора.

3. Принципы работы системы ALADDIN

  • Принцип анонимности — система не собирает, не обрабатывает и не хранит персональные данные пользователей
  • Принцип локальной обработки — все данные, которые могут содержать персональную информацию, обрабатываются исключительно на устройстве пользователя
  • Принцип обезличивания — все данные, передаваемые в систему, автоматически обезличиваются
  • Принцип минимальной необходимости — система собирает только те данные, которые необходимы для обеспечения функциональности
  • Принцип No-Logs — система не ведет логи о том, какие сайты посещают пользователи, какой контент они просматривают

4. Аутентификация и регистрация

Анонимная регистрация:

  • Создание анонимного профиля с указанием роли (родитель/ребенок/пожилой)
  • Указание возрастной группы без конкретного возраста
  • Регистрация устройств по типу (смартфон/планшет/компьютер)
  • НЕ требуется: имя, фамилия, email, телефон, адрес, дата рождения

Локальная аутентификация:

  • Использование PIN-кода или пароля (хранится локально на устройстве)
  • Биометрическая аутентификация (отпечаток, Face ID) — обрабатывается только на устройстве
  • QR-код для входа в семейную группу
  • НЕ передается на серверы: биометрические данные, пароли, PIN-коды

Агрегированная аналитика:

  • Общее количество подключений по типам
  • Статистика использования серверов
  • Эффективность защиты
  • Тренды безопасности

8. Меры защиты данных

Технические меры:

  • Шифрование всех передаваемых данных (AES-256)
  • Анонимизация данных на этапе сбора
  • Локальное хранение чувствительной информации
  • Регулярное обновление системы безопасности
  • No-Logs политика (отсутствие логов пользовательской активности)

Организационные меры:

  • Обучение персонала принципам анонимности
  • Регулярный аудит системы на предмет утечек
  • Контроль доступа к данным
  • Политика "нулевой толерантности" к сбору персональных данных

  • Агрегированная аналитика: 1 год
